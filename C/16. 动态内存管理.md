#  2.  为什么要学动态内存管理?

我们平常使用数组时:

```c
int arr[6]={1,2,3,4,5,6};
```

这是在栈区开辟的空间
==并且它的大小是固定的,被初始化成6==

`原因一:`
>**很多情况下,我们需要的空间大小
在程序运行后才知道,此时数组不管用了
只能尝试使用动态内存管理的方式了**!

`原因二:`

>**在做题时经常遇见需要手动动态
开辟空间的题目,且在笔试中也经常考察**

#  3. 动态内存管理函数介绍

==动态内存管理有四个函数:==

<font color = blue>1. malloc </font>
<font color = blue>2. calloc</font>
<font color = blue>3. realloc</font>
<font color = blue>4. free </font>

`这些函数定义在头文件: stdlib.h中`

**前两个是用于开辟空间的
realloc是用于扩容的
而free是释放空间的**

#  4. malloc函数详细介绍
先看它在[cplusplus](https://legacy.cplusplus.com/reference/)上的定义:

![在这里插入图片描述](https://i-blog.csdnimg.cn/blog_migrate/6d6f86a1cb805554a06cf1fe7f1cea8e.png)


==我直接把要点翻译过来:==

- <font color = gree>功能:它向内存申请一块连续可用的空间</font>

- <font color = gree>申请空间成功就返回指向这块空间的指针</font>

- <font color = gree>申请空间失败就返回空指针NULL</font>

- <font color = gree>返回类型是void*
它可以申请任一类型的空间</font>

- <font color = gree>参数size的单位是字节
开辟size个字节的空间</font>


==需要注意的地方:==
>1. 它的返回类型是void*,你在开辟空间时
>应强制类型转换成你要开辟的空间的类型

```c
struct S
{
  int a;
  char cc;
  double d;
};

int* p = (int*)malloc(40);
char* arr = (char*)malloc(10);
struct S* ss = (struct S*)malloc(20);
```

`总结:开辟什么类型的空间就强转成什么`

***

>2. malloc有可能申请空间失败
>所以每次开辟空间后要判断指针是否为空
>若指针为空应该直接终止掉程序

```c
int* p = (int*)malloc(40);
if(p == NULL)
{
	printf("动态申请失败");
	exit(-1)//退出程序
}
else //不为空再去使用它
{
	//......
}
```

`总结:每次开辟空间后要判断空指针`

#  5. free函数详细介绍
**先看free在官网上的定义:**

![在这里插入图片描述](https://i-blog.csdnimg.cn/blog_migrate/05f4c1e35d16462a9fb97b3a8603c0af.png)

==将要点翻译过来:==

- <font color = gree>free用于释放动态开辟的空间</font>

- <font color = gree>参数ptr指向的空间必须是动态开辟的</font>

- <font color = gree>若ptr是空指针,则free函数什么都不做</font>

==需要注意的地方:==

>1. free释放的空间必须在堆区
>也就是必须是动态开辟的内存

```c
//堆区开辟的空间
int* p = (int*)malloc(40);
//栈区开辟的空间
int arr[4] = {1,2,3,4};
//正确的用法
free(p);
//错误的用法
free(arr);
```

`总结:free释放的空间必须在堆区`

***
>2. 被free释放后的空间就还给操作系统了
>但是指针还是指向已经释放的空间
>所以在free释放掉一块空间后
>还要将指针赋值为空才安全

```c
int* p = (int*)malloc(40);
free(p);
p = NULL;
```

`总结:释放掉空间后记得要置空`

#  6. calloc函数详细介绍
先看calloc的定义:

![在这里插入图片描述](https://i-blog.csdnimg.cn/blog_migrate/42a45db598e421da0ac873b7a6d9b8c2.png)

==重点翻译:==

- <font color = gree>参数num是要开辟的空间个数</font>

- <font color = gree>参数size是每份空间所占字节数</font>

- <font color = gree>calloc不仅会开辟空间
还将所有数据初始化为0</font>

- <font color = gree>它和malloc的区别就是前者会初始化 </font>


==使用举例:==

```c
//开辟10个整型空间
int* p = (int*)calloc(10,sizeof(int));
if(p == NULL)
{
	printf("开辟失败");
	exit(-1);
}
else
{
	//......
}
```

`可以在不同场景选择使用`
`malloc或者calloc`

#  7. realloc函数详细介绍
先看realloc的定义:

![在这里插入图片描述](https://i-blog.csdnimg.cn/blog_migrate/7eb27e3de3fe2181a52a499c72f3401d.png)

==重点翻译:==

- <font color = gree>参数ptr是指向要扩容的空间的指针</font>
- <font color = gree>参数size是扩容后空间总大小
单位是字节</font>
- <font color = gree>返回值为扩容之后的内存起始位置</font>
- <font color = gree>此函数在调整原先空间大小的基础上
还会将原先空间的内容拷贝过来</font>



==需要注意的地方:==

>1. realloc扩容的空间必须在堆区

```c
int* p = (int*)malloc(40);
int arr[4]={1,2,3,4};
//正确用法
int* pp = (int*)realloc(p,60);
//错误用法
int* parr = (int*)realloc(arr,30);
```

***
>2. 若参数ptr是空指针NULL
>则realloc的功能相当于malloc

```c
int* p = NULL;
int* pp = (int*)realloc(p,40);
```

`此时的realloc相当于malloc`

***
>3. realloc扩容后,原先指向
>这块空间的指针会自动置空
>不需要再手动置空

```c
int* p = (int*)malloc(40);
int* pp = (int*)realloc(p,60);

free(pp);
pp=NULL;
```

`总结:realloc扩容后,原先的指针可以不管`
